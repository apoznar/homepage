Wiedza związana z technologiami frontendowymi

Różne ciekawostki

Kombinacja alt+shift+F automatycznie czyści kod w Visual Studio Code. Wykrzyknik oraz tabulator automatycznie wypełnia sekcję head. Metoda Kanban (Todo, Doing - jedna rzecz, Done) wykorzystywana jest w Trello. Pamiętaj o sposobach zapisu - camelCase, kebab-case, snake_case, PascalCase. Kody odpowiedzi serwera: 200 to normalny odczyt strony, 30x to przekierowania, 404 to brak strony, 50x to błąd wewnętrzny serwera. Musimy pamiętać o roocie(/) w ścieżce do zasobu (jeżeli strona jest postawiona na zewnętrznym serwerze).

Terminal ma na Windowsie formę wiersza poleceń (cmd). Komenda dir pokazuje spis plików w folderze, cd przechodzi do wskazanego pliku (tab podpowiada nazwę pliku), mkdir nazwa tworzy folder, code . uruchamia vsc. cd .. przenosi do folderu wyżej w strukturze.

Ciekawe rozszerzenia do VSC - Prettier (formater kodu) oraz Live Server (podgląd dynamiczny strony, dzięki ip config możemy dostać ip i podejrzeć plik z innej lokalizacji).

GIT & GitHub

System kontroli wersji. Komendy w bashu: git --version (sprawdza wersję), git config --global user.name "Jan Nowak" (nadaje nazwę użytkownika), git config --global user.email jannowak@example.com (nadaje maila uzytkownika). Git config --get user.name sprawdza poprawność. Git init inicjalizuje repozytorium. Git status pokazuje status projektu. Git add . dodaje do śledzenia wszystkie pliki w repozytorium. Git commit -m "initial commit" zatwierdza zmiany z komentarzem - warto commitować często. Gitk pokazuje historię commitów. Clear czyści terminal. Zainstalowane rozszerzenie Git Lens. Na końcu piszemy git push aby przesunąć zmiany na GitHuba. Możemy też uwidocznić stronę na GitHub Pages poprzez ustawienia repozytorium, zmieniamy source na branch i używamy linka powyżej.

Hamburger Menu

Potrzeba nawigacji, styl do nawigacji na mobile, na desktop, przycisk do pokazywania/ukrywania (zmiana klasy), kod js do podpięcia się pod przycisk i kod js do zmiany klasy. Nadawanie i usuwanie klasy - element.classList.add/remove/toggle/contains[wyrażenie logiczne]('klasa-z-js').Rozpoczynając od początku mamy element nav, potem np. zmieniamy atrybuty obrazka na max-width: 100% i display blocki robimy inne działania dla responsywności mobilnej elementów. Tworzymy przycisk ukrywany pod hamburgerem z prawej strony i na końcu selektor klasy hamburger i event listener powiązany z hamburgerem który uwidacznia nam nav oraz toggle na navigation--open. Jest też atrybut do button aria-label do opisywania przycisku.
